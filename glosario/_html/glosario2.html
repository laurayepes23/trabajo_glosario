A — Algoritmo

Secuencia ordenada y finita de pasos que permiten resolver un problema específico o realizar una tarea determinada. Son la base de la programación y permiten estructurar soluciones lógicas que pueden ser interpretadas por un ordenador.

B — Back-End

Parte de una aplicación que gestiona la lógica de negocio, bases de datos y operaciones del servidor. Aunque no es visible para el usuario, es esencial para que la aplicación funcione correctamente, procesando datos y enviando respuestas al Front-End.

C — Compilador

Programa que traduce código fuente escrito en un lenguaje de alto nivel a lenguaje máquina. Este proceso permite que un ordenador ejecute el software de forma eficiente y detecta errores en tiempo de compilación.

D — Depuración

Proceso de identificar, aislar y corregir errores o fallos en el código de una aplicación. Herramientas de depuración permiten a los desarrolladores analizar el flujo de ejecución y las variables para solucionar problemas.

E — Entidad

Representación de un objeto, concepto o elemento del mundo real dentro de un sistema de base de datos, que posee atributos y se relaciona con otras entidades a través de relaciones definidas en el modelo Entidad-Relación.

F — Framework

Estructura conceptual y tecnológica predefinida que facilita el desarrollo de software al proporcionar herramientas, bibliotecas y buenas prácticas. Permite a los desarrolladores centrarse en resolver problemas específicos sin preocuparse por aspectos generales.

G — Git

Sistema de control de versiones distribuido que permite a múltiples desarrolladores trabajar en un mismo proyecto, gestionando cambios, ramificaciones y fusiones de código de manera eficiente y segura.

H — HTML

Lenguaje de marcado utilizado para estructurar el contenido de páginas web. Permite definir elementos como encabezados, párrafos, enlaces, imágenes y otros recursos multimedia que componen la interfaz de usuario en internet.

I — Iteración

Repetición de un conjunto de instrucciones o procesos hasta cumplir una condición establecida. Las iteraciones son fundamentales en algoritmos y bucles de programación, permitiendo automatizar tareas repetitivas.

J — JSON

Formato ligero de intercambio de datos basado en texto, fácil de leer y escribir para humanos y sencillo de interpretar por máquinas. Es ampliamente utilizado para comunicar datos entre aplicaciones cliente-servidor.

K — Keylogger

Software malicioso que registra las pulsaciones de teclado de un usuario, generalmente con intenciones de obtener información sensible como contraseñas o datos bancarios sin su consentimiento.

L — Librería

Conjunto de funciones y recursos predefinidos que los desarrolladores pueden utilizar para evitar reescribir código. Facilitan tareas específicas como manejo de fechas, conexiones a bases de datos o generación de gráficos.

M — Modelo de Datos

Representación estructurada de los datos utilizados en una aplicación, indicando su organización, relaciones y restricciones. Es fundamental en el diseño de bases de datos para garantizar la integridad y coherencia de la información.

N — Node.js

Entorno de ejecución para JavaScript del lado del servidor, basado en el motor V8 de Google Chrome. Permite desarrollar aplicaciones escalables y de alto rendimiento, especialmente en entornos web y tiempo real.

O — Open Source

Software cuyo código fuente está disponible para ser modificado, distribuido y utilizado libremente por cualquier persona. Fomenta la colaboración, la transparencia y la innovación en el desarrollo de aplicaciones.

P — Prototipo

Versión preliminar de un sistema o aplicación que permite evaluar su funcionalidad, diseño y usabilidad antes del desarrollo final. Los prototipos ayudan a detectar errores conceptuales y a mejorar la experiencia de usuario.

Q — Query

Instrucción utilizada para consultar, insertar, actualizar o eliminar datos de una base de datos. Las consultas permiten obtener información específica mediante lenguajes como SQL.

R — Repositorio

Almacén centralizado o distribuido donde se guarda y gestiona el código fuente de un proyecto, junto con su historial de cambios. Facilita la colaboración y control de versiones en entornos de desarrollo de software.

S — Scrum

Metodología ágil para la gestión de proyectos de software que se basa en iteraciones llamadas sprints, promoviendo la comunicación, flexibilidad y entrega continua de valor al cliente.

T — Testing

Proceso de verificación y validación que permite asegurar que un software cumple con los requisitos especificados y funciona correctamente. Incluye pruebas de funcionalidad, rendimiento, seguridad y usabilidad.

U — UML

Lenguaje de modelado unificado que permite representar gráficamente el diseño y estructura de un sistema de software mediante diagramas de clases, casos de uso, secuencia, entre otros.

V — Variable

Espacio de almacenamiento en memoria identificado por un nombre, que contiene un valor que puede cambiar durante la ejecución de un programa. Son esenciales para manejar datos dinámicos en aplicaciones.

W — Web Service

Conjunto de protocolos y estándares que permiten la comunicación entre aplicaciones a través de la web, independientemente de las plataformas o lenguajes de programación en que estén desarrolladas.

X — XML

Lenguaje de marcado que permite estructurar, almacenar e intercambiar información de forma legible tanto para humanos como para máquinas. Se utiliza ampliamente en configuraciones, servicios web y transmisión de datos.

Y — YAML

Formato de serialización de datos legible por humanos, utilizado para almacenar configuraciones y estructuras de datos de forma simple y jerárquica. Es común en entornos DevOps y despliegue de aplicaciones.

Z — Zero-Day

Vulnerabilidad desconocida por los desarrolladores de un software que puede ser explotada antes de que exista una solución. Son riesgos críticos para la seguridad de sistemas y redes.
